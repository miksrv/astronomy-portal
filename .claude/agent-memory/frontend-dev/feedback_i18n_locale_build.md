---
name: i18n locale build process
description: How translation keys are managed — scanner reads source files, do not edit locale JSON directly
type: feedback
---

Translation keys are auto-generated by `yarn locales:build` (runs `i18next-scanner`). The scanner reads `t('key', 'default')` calls from all `.ts`/`.tsx` files and updates `/public/locales/{ru,en}/translation.json`.

**Why:** Editing locale JSON directly leads to keys being removed on the next `locales:build` run, which removes unused keys.

**How to apply:** Always add `t('key', 'defaultValue')` in the source file, then run `yarn locales:build` to generate the locale entries. Never hand-edit the JSON. The Russian `defaultValue` becomes the `ru` translation; the `en` file gets the same default and should be manually translated if needed.
