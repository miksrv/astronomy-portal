---
name: i18n locale build process
description: How translation keys are managed — scanner reads source files, do not edit locale JSON directly
type: feedback
---

Translation keys are auto-generated by `yarn locales:build` (runs `i18next-scanner`). The scanner reads `t('key', 'default')` calls from all `.ts`/`.tsx` files and updates `/public/locales/{ru,en}/translation.json`.

**Why:** Editing locale JSON directly is safe for `en/translation.json` because the scanner config now uses a custom `flush` that preserves existing EN values. However, never hand-edit `ru/translation.json` values — the scanner overwrites `ru` from source `defaultValue` strings.

**How to apply:**
- Add `t('key', 'Russian default')` in source, then run `yarn locales:build`. The Russian value goes into `ru`; new keys in `en` get empty string `""`.
- To add an English translation: edit `en/translation.json` directly (fill in the `""` value), then run `yarn locales:build` again — the scanner will preserve the non-empty EN value.
- The scanner config (`i18next-scanner.config.js`) has a custom `flush` function that snapshots `en/translation.json` at module load time and merges the snapshot back over the scanner output, so non-empty EN values are never overwritten. New keys always get `""` in EN.
- `ru/translation.json` is always written verbatim from the scanner (Russian source defaults).
